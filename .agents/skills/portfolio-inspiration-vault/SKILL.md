## 📐 Layout Principles (Catalog vs. Blog)

To avoid a "flat blog" aesthetic, every layout must use **Asymmetric Grid Shifts** and **Bento Grid Blocks**:

### 1. Asymmetric Layout Splits
- **Never** stack content in single-column text blocks.
- **Always** separate layout cards into grid columns (e.g. 12-column layouts where 4 columns host structural log metadata, 8 columns host headlines, and visual assets overlap rules).
- Use thin hairline rules (`border-border/30`) to establish clear alignment lines.

### 2. Bento Grid System
- Arrange related statements, metrics, and domain blocks into a unified Bento Grid.
- Give each block a distinct height/width ratio (e.g. a wide 8-col block for the core thesis next to a compact 4-col block foraltimeter metrics).
- Corner nodes must contain blueprint crosshair markers (`+`, `⌞ ⌟`) to establish a precision aesthetic.

### 3. Typography as Art
- Use extreme scale contrast (e.g., 72px serif headers paired with 10px uppercase monospace labels).
- Integrate high-character italicized keywords inside display headlines (e.g. *"systems for* **_consequential decisions_***"*).

## 🎬 GSAP Kinetic Motion Rules

Animations must represent **cockpit feedback responses**:
- **Altimeter counts:** Numbers roll up from 0 to their targets with a `power3.out` ease.
- **Glides:** Layout blocks fade and glide up in staggered sequence as the user scrolls.
- **Hover crosshairs:** Mouse hover triggers coordinate markers and frame guidelines to transition from low-contrast opacity (`opacity-20`) to high-visibility teal (`opacity-80`).
